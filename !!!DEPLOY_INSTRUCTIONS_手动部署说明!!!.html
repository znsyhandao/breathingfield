🧘 冥想呼吸场应用 - 手动部署说明

请按照以下步骤，通过 GitHub 网页界面 上传本文件夹内的所有文件。

第1步：打开你的仓库

访问： https://github.com/znsyhandao/breathingfield

第2步：切换到 gh-pages 分支

在仓库页面，点击分支下拉菜单（通常显示为 main ▼）。

在输入框中键入 gh-pages 并按下回车。

• 如果分支已存在，页面会切换过去。

• 如果分支不存在，GitHub 会显示 “Branch not found” 并提供一个 “Create branch: gh-pages from main” 的按钮，点击它来创建。

第3步：删除旧文件（重要！）

切换到 gh-pages 分支后，点击 “Add file” 按钮，然后选择 “Upload files”。

在打开的上传页面，先不要上传新文件。

将页面向下滚动，你会看到当前分支的所有文件列表。

在每个文件左侧打勾（或使用顶部的全选复选框），然后点击上方的 “Delete” 按钮，删除所有现有文件。在确认弹窗中提交删除操作。

这一步是为了确保没有残留的旧文件（比如 .mp3 文件）干扰。

第4步：上传新文件

完成删除后，页面会刷新。再次点击 “Add file” -> “Upload files”。

打开文件资源管理器，导航到包含此说明文件的文件夹：
C:\Users\[你的用户名]\Desktop\BreathingField_For_GitHubPages

选中此文件夹内的所有文件（Ctrl+A），然后将它们拖拽到GitHub网页的上传区域。

上传的文件应包含：index.html, 404.html, .nojekyll, assets/ 文件夹, audio/ 文件夹等。

第5步：提交更改

在页面底部的提交更改区域：

  • 在第一个输入框填写：Manual deploy: Fixed audio files
  • 第二个输入框可选填。
  • 确保选择 “Commit directly to the gh-pages branch.”

点击绿色的 “Commit changes” 按钮。

第6步：等待并访问

提交后，GitHub 需要 1-3 分钟 来更新网站。

之后，访问你的在线应用：

https://znsyhandao.github.io/breathingfield/

在手机上访问时，请下拉刷新并选择“强制重新加载”以清除缓存。
